# 🔍 Retrieval Agent

**Node:** `retriever_node`  
**File:** [`retriever_node.py`](../nodes/retriever_node.py)  
**Role:** Semantic search against Pinecone vector store  

## Purpose

Executes each search query generated by the Planner against the Pinecone vector store using **MMR (Maximal Marginal Relevance)** retrieval. This ensures diverse, non-redundant document chunks are returned.

## How It Works

1. Takes the `plan` (list of search queries) from the Planner
2. Runs all queries concurrently via `asyncio.gather()`
3. Deduplicates results using content-hash based `doc_id` (SHA-1)
4. Formats citations for each unique document chunk
5. Applies user-scoped filtering via `user_id` metadata

## Key Features

- **Concurrent execution**: All search queries run in parallel (15s timeout each)
- **Deduplication**: SHA-1 hash of `filename + chunk_index + content[:100]`
- **User scoping**: Only retrieves documents belonging to the authenticated user
- **Graceful fallback**: If `requires_context=False`, returns empty results immediately

## State Updates

| Field | Value |
|-------|-------|
| `retrieved_docs` | List of `{page_content, metadata, doc_id}` dicts |
| `citations` | Formatted citation objects for the frontend |
| `current_node` | `"retriever"` |

## Configuration

- **Top-K:** `DEFAULT_TOP_K` from constants (default: 6)
- **Retrieval strategy:** MMR (diversity-optimized)
- **Timeout:** 15 seconds per query
